# NFramework.Core.CLI

A compile-time-first CLI application framework for .NET 11+ that enables developers to build command-line interfaces using declarative attributes and source-generated boilerplate.

[![NuGet](https://img.shields.io/nuget/v/NFramework.Core.CLI.Abstractions)](https://www.nuget.org/packages/NFramework.Core.CLI.Abstractions/)
[![NuGet](https://img.shields.io/nuget/v/NFramework.Core.CLI.SpectreConsoleUI)](https://www.nuget.org/packages/NFramework.Core.CLI.SpectreConsoleUI/)

## Features

- **Zero Boilerplate**: Define commands with attributes; source generator handles the rest
- **Compile-Time Safety**: Catch command definition errors at build time
- **Testable Abstractions**: Abstract terminal session interface for unit testing
- **Rich Terminal UI**: Text input, selection prompts, and formatted tables
- **Native AOT Compatible**: No runtime reflection or dynamic code generation
- **DI Integration**: Seamless integration with Microsoft.Extensions.DependencyInjection

## Installation

```bash
dotnet add package NFramework.Core.CLI.Abstractions
dotnet add package NFramework.Core.CLI.SpectreConsoleUI
dotnet add package NFramework.Core.CLI.SpectreConsoleUI.Generators
```

## Quick Start

### 1. Define Your Application

```csharp
using NFramework.Core.CLI.Abstractions;

namespace MyCli;

[CliApplication("mycli")]
public sealed partial class Application : ICliApplication
{
    // The Run() method is generated by the source generator
}
```

### 2. Define Settings

```csharp
namespace MyCli;

public sealed record Settings
{
    [CliOption("-v|--verbose")]
    [Description("Enable verbose output")]
    public bool Verbose { get; init; }

    [CliOption("-o|--output")]
    [Description("Output file path")]
    public string? OutputPath { get; init; }
}
```

### 3. Define Commands

```csharp
namespace MyCli.Commands;

[CliCommand("greet", "Prints a friendly greeting")]
[CliExample("greet", "--verbose", "World")]
public sealed class GreetCommand : IAsyncCliCommand<Settings>
{
    [CliArgument(0, "name")]
    [Description("The name to greet")]
    public string Name { get; init; } = "World";

    public Task<int> ExecuteAsync(
        CliCommandContext context,
        Settings settings,
        CancellationToken cancellationToken)
    {
        if (settings.Verbose)
        {
            Console.WriteLine($"Executing greet command with argument: {Name}");
        }

        Console.WriteLine($"Hello, {Name}!");
        return Task.FromResult(0);
    }
}
```

### 4. Run Your CLI

```csharp
using Microsoft.Extensions.DependencyInjection;
using NFramework.Core.CLI.Abstractions;
using NFramework.Core.CLI.SpectreConsoleUI.DependencyInjection;

namespace MyCli;

static class Program
{
    static async Task<int> Main(string[] args)
    {
        var services = new ServiceCollection()
            .AddCoreCliSpectreConsoleUi()
            .BuildServiceProvider();

        var application = new Application();
        return application.Run(args);
    }
}
```

## Terminal Session Abstraction

The `ITerminalSession` interface provides testable abstractions for terminal interactions:

```csharp
public interface ITerminalSession
{
    bool IsInteractive { get; }

    Task<TerminalTextInputResult> PromptForTextAsync(
        TerminalTextPrompt prompt,
        CancellationToken cancellationToken);

    Task<TerminalSelectionResult> PromptForSelectionAsync(
        TerminalSelectionPrompt prompt,
        CancellationToken cancellationToken);

    void WriteLine(string message);
    void WriteErrorLine(string message);
    void RenderTable(TerminalTable table);
}
```

### Text Input Prompts

```csharp
var result = await _terminal.PromptForTextAsync(
    new TerminalTextPrompt(
        "Enter your name:",
        "Name is required"
    ),
    cancellationToken
);

if (result.WasCancelled)
{
    return 1;
}

var name = result.Value!;
```

### Selection Prompts

```csharp
var result = await _terminal.PromptForSelectionAsync(
    new TerminalSelectionPrompt(
        "Choose an option:",
        [
            new TerminalSelectionOption("1", "Option One"),
            new TerminalSelectionOption("2", "Option Two"),
            new TerminalSelectionOption("3", "Option Three"),
        ]
    ),
    cancellationToken
);

if (result.WasCancelled)
{
    return 1;
}

var selected = result.SelectedValue;
```

### Table Rendering

```csharp
_terminal.RenderTable(
    new TerminalTable(
        ["Name", "Version", "Description"],
        [
            new TerminalTableRow(["nfw", "1.0.0", "NFramework CLI"]),
            new TerminalTableRow(["core-cli", "1.0.0", "Core CLI abstractions"]),
        ]
    )
);
```

## Attributes Reference

### `[CliApplication(name)]`

Marks the application entry point class.

```csharp
[CliApplication("myapp")]
public sealed partial class Application : ICliApplication { }
```

- `name`: The application name (used in help text and error messages)
- `ValidateExamples`: Whether to validate example attributes (default: `true`)

### `[CliCommand(name, description)]`

Defines a command.

```csharp
[CliCommand("init", "Initialize a new workspace")]
public class InitCommand : IAsyncCliCommand<Settings> { }
```

- `name`: The command name (must be a valid CLI identifier)
- `description`: Help text description

### `[CliArgument(position, valueName)]`

Defines a positional argument.

```csharp
[CliArgument(0, "path")]
public string Path { get; init; } = string.Empty;
```

- `position`: Zero-based position (must be unique and non-negative)
- `valueName`: Display name for help text
- `IsRequired`: Whether the argument is required (default: `false`)

### `[CliOption(template)]`

Defines a named option/flag.

```csharp
[CliOption("-f|--force")]
[CliOption("-v|--verbose", IsHidden = true)]
public bool Verbose { get; init; }
```

- `template`: Spectre.Console option template (e.g., `-v|--verbose`)
- `IsRequired`: Whether the option is required (default: `false`)
- `IsHidden`: Hide from help text (default: `false`)
- `ValueIsOptional`: Option value can be omitted (default: `false`)

### `[CliExample(params string[])]`

Adds usage examples to help text.

```csharp
[CliExample("init", "my-workspace")]
[CliExample("init", "my-workspace", "--template", "blank")]
public class InitCommand : IAsyncCliCommand<Settings> { }
```

### `[Description]`

Adds descriptions to properties (from `System.ComponentModel`).

```csharp
[Description("The output directory path")]
public string? OutputPath { get; init; }
```

## Advanced Usage

### Nested Commands

Commands can be nested by using the `.` separator in the command name:

```csharp
[CliCommand("database.migrate", "Run database migrations")]
public class MigrateCommand : IAsyncCliCommand<Settings> { }

[CliCommand("database.rollback", "Rollback last migration")]
public class RollbackCommand : IAsyncCliCommand<Settings> { }
```

### Testing Commands

Use the `ITerminalSession` abstraction to test commands without a terminal:

```csharp
using Moq;
using Xunit;

public class GreetCommandTests
{
    [Fact]
    public async Task ExecuteAsync_WritesGreeting()
    {
        // Arrange
        var terminal = new Mock<ITerminalSession>();
        var command = new GreetCommand { Name = "Test" };
        var context = new CliCommandContext("greet", []);
        var settings = new Settings();

        // Act
        var result = await command.ExecuteAsync(context, settings, default);

        // Assert
        Assert.Equal(0, result);
        terminal.Verify(t => t.WriteLine("Hello, Test!"), Times.Once);
    }
}
```

### Dependency Injection

Inject services into your commands through the settings type or constructor:

```csharp
public class Settings
{
    public required ILogger Logger { get; init; }
}

public class GreetCommand : IAsyncCliCommand<Settings>
{
    public Task<int> ExecuteAsync(
        CliCommandContext context,
        Settings settings,
        CancellationToken cancellationToken)
    {
        settings.Logger.LogInformation("Greeting {Name}", Name);
        // ...
    }
}
```

| Code | Meaning                 |
| ---- | ----------------------- |
| 0    | Success                 |
| 1    | General error           |
| 2    | Invalid arguments       |
| 130  | User cancelled (Ctrl+C) |

```bash
dotnet publish -c Release -r win-x64 --self-contained
dotnet publish -c Release -r linux-x64 --self-contained
dotnet publish -c Release -r osx-x64 --self-contained
```
